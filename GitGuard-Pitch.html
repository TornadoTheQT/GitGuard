The context

More agents at the helm. Fewer hands on the wheel.

In agentic coding workflows, teams increasingly let AI agents edit, refactor, and ship without reviewing every line. That trade-off makes sense — until a credential lands in a commit. Models are powerful editors, but they are not absolute guardrails against leaking sensitive information.

Shift

Less micromanagement

Developers hand off boilerplate, refactors, and even security fixes to coding agents. The workflow is faster; oversight is thinner.

Gap

Models aren't vaults

An agent can paste a live key into config, echo a token in a snippet, or miss a secret already buried in git history. No model promise replaces deterministic detection.

Stakes

Minutes, not days

Once exposed, keys are scraped fast. You need offline scanning and a fix path you control — not blind trust in the agent that wrote the leak.

Our addition

Scanning is necessary. The harness is the spin.

Secret scanning belongs in every serious pipeline — GitGuard does that offline, with context-aware scoring and full git-history coverage. What we added on top is a custom local fix harness: GitGuard prepares the task, sandboxes the target, and runs your chosen model inside strict boundaries. You review every diff before anything touches your repo.

  • ›Baseline, done right. Four correlated detectors, no provider-side validation, no exfiltration. The scanner your agents can't replace.
  • ›Harness, not autopilot. --fix copies into a temp workspace, invokes your configured agent once, and stops. No commits, no pushes, no edits outside the sandbox.
  • ›Model of your choice. Pass --model provider/model or set a default at setup. We run Claude Sonnet 4.6 — swap it anytime.

The clever part · context scoring

It reads the room, not just the regex.

A raw match isn't a verdict. GitGuard scores the words around every hit, so real keys rise and noise sinks.

↑ Promote

Raise severity

Proximity to password, live, production, or a .env/config file lifts severity and confidence.

↓ Demote

Sink the noise

Markers like placeholder, example.com, CHANGE_ME drop a finding toward INFO instead of crying wolf.

≈ Keep

Surface, don't hide

Placeholders are reported as INFO, so reviewers see intent without the alert fatigue. Recall and precision, together.

Why offline matters

Deterministic detection. No trust-the-model scanning.

Agents may suggest fixes; GitGuard verifies leaks with rules, not LLM guesswork. Your secrets never leave the machine during a scan.

The fix harness

Agent-assisted remediation, bounded and reviewable.

--ai exports a Markdown brief if you want to remediate elsewhere. --fix uses GitGuard's custom local harness — a disposable workspace, a structured task contract, and a model you configure (default: claude-sonnet-4-6 via --model).

01

Scan & isolate

Findings become a remediation brief. GitGuard copies the target into a temp workspace — your tree stays untouched.

02

Harness + model

The configured agent edits only inside the sandbox. One pass, smallest safe changes. No git commits, no network exfiltration.

03

Diff & review

GitGuard computes the unified diff, validates paths, and presents it for your review — not the agent's.

04

Apply on consent

Only after you confirm: apply locally, write a .fixed.zip, or save a patch. Optional --test-cmd verifies the result.

● Human-in-the-loop gate.  The harness enables the model; GitGuard keeps authority over scan, sandbox, diff, and apply.
Severity & risk

A score that means something.

Five ordered levels feed a single 0 to 100 score, weighted by severity, scaled by confidence, with extra weight for history and config or prod files.

CRITICAL · +35
HIGH · +20
MED · +10
Critical +35 High +20 Medium +10 Low +3

--fail-on HIGH turns the score into a CI gate, so the build fails the moment a HIGH or CRITICAL secret lands.

Install

One command

install.sh builds the CLI; --with-fix-agent installs the harness runtime and configures your default model.

setup-fix-agent

Pick your model

Installs the OpenClaw harness, sets claude-sonnet-4-6 by default, and checks auth. Override anytime with --model.

CI

Native gate

--fail-on HIGH blocks risky merges. Export JSON, CSV, or an --ai brief for agents that live outside GitGuard.

doctor

Self-diagnosing

Checks Python, git, harness readiness, permissions, and whether a target is scannable before you waste a fix pass.
